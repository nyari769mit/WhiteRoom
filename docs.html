How it works
The Universal Cycle

Every agent — whether solo or part of a fleet — follows the same governance cycle. Inspired by maritime watch systems.

WORK

Agent works normally. WhiteRoom tracks tokens, tasks, time.

→

LIMIT HIT

Watch limit reached. WhiteRoom blocks further calls (429).

→

COMPRESS

Context compressed into structured handover document.

→

REST

Context cleared. Agent rests. Health recovers.

→

RESTART

Agent resumes with only the compressed handover. Far fewer tokens.

1

Agent calls WhiteRoom instead of the LLM directly

WhiteRoom sits as a transparent proxy. The agent doesn't know it's being governed — it just makes normal API calls. WhiteRoom intercepts every request at the /v1/messages endpoint.

TRANSPARENT PROXY
2

WhiteRoom tracks every call

Tokens used, tasks completed, minutes worked — all tracked per agent, per fleet. The audit log captures every event with timestamps. No instrumentation needed in your agent code.

AUTOMATIC TRACKING
3

Watch limit triggers auto-compression

When an agent exceeds its watch limit, WhiteRoom automatically calls the LLM one final time with a compression prompt. The agent's full task history is distilled into a structured handover document covering decisions made, current state, pending tasks, and warnings.

AUTO-COMPRESSION
4

Relief agent or self-restart with compressed context

For paired fleets: the compressed handover is passed to the relief agent. For solo agents: the agent restarts with only the handover summary as its starting context. Either way, the next session costs a fraction of the tokens.

CONTEXT RESET
5

Auto-alarm wakes the agent after rest

WhiteRoom runs a background timer. When the rest period ends, it fires the alarm automatically. The agent resumes without any manual intervention. The full cycle repeats indefinitely.

FULLY AUTOMATIC
Quick Start
Connect in 60 seconds

No account. No installation. One environment variable.

1
Point your agent at WhiteRoom

# Anthropic
export ANTHROPIC_BASE_URL=\
  https://whiteroom-m4j4.onrender.com

# OpenAI / compatible
export OPENAI_BASE_URL=\
  https://whiteroom-m4j4.onrender.com

Your existing API key and all your code stays the same. WhiteRoom intercepts calls automatically.

2
Run your agent as normal

# Python example
from anthropic import Anthropic
client = Anthropic()  # uses env var

# Node.js example
const anthropic = new Anthropic();
// uses ANTHROPIC_BASE_URL automatically

No SDK changes. No wrapper code. WhiteRoom is invisible to your agent.

3
Open your personalized dashboard

https://whiteroom-m4j4.onrender.com

Enter your API key on the login screen. WhiteRoom identifies your fleet automatically. Your key is hashed immediately — never stored raw.

Click RUN LIVE. Your agent appears within 10 seconds.

4
Optional: register a paired fleet

curl -X POST \
  https://whiteroom-m4j4.onrender.com\
  /api/white-room \
  -d '{"action":"register_agent",
       "fleet_id":"my-fleet",
       "agent_id":"primary",
       "agent_role":"Researcher"}'

Pairing enables handover between primary and relief agents for continuous operation.

Features
Everything WhiteRoom does

Governance at the infrastructure layer. No framework required.

🔒

Watch Limit Enforcement

Agents are blocked with a 429 response when they exceed their watch limit. No agent can bypass governance — it's enforced at the proxy layer.

CORE GOVERNANCE
🗜️

Auto Context Compression

When a watch limit is hit, WhiteRoom automatically generates a structured handover document using the LLM itself — decisions, state, pending tasks, warnings.

AUTO-COMPRESSION
💤

Mandatory Rest Periods

After handover, agents enter a rest period. All calls are blocked until rest is complete. WhiteRoom auto-fires the alarm when rest ends — no manual intervention.

REST ENFORCEMENT
🔄

Automatic Handover

When a primary agent hits its limit, WhiteRoom automatically triggers handover to the paired relief agent. The compressed context is passed seamlessly.

PAIRED FLEETS
🧍

Single Agent Mode

No relief agent needed. The agent compresses its own context, rests, then restarts fresh with only the handover summary. Same savings, solo operation.

SELF-HANDOVER
📊

REAL-TIME

API Key Privacy

PRIVACY FIRST

API Reference
Governance endpoints

All calls go to POST /api/white-room with a JSON body.

ACTION REQUIRED FIELDS DESCRIPTION
register_agent fleet_id, agent_id
agent_role, llm_endpoint		 Register an agent with a fleet. Pass llm_endpoint to use a non-Anthropic LLM.
start_watch fleet_id, agent_id Start the watch timer for an agent. Must be called before the agent begins work.
check_watch fleet_id, agent_id Get current watch status — minutes worked, tokens used, % complete, needsHandover flag.
complete_task fleet_id, agent_id, task_name
minutes_spent, tokens_used		 Log a completed task. Called automatically by WhiteRoom's proxy after each LLM call.
pair_agents fleet_id, agent_id, paired_with Pair a primary agent with its relief agent for automatic handover.
initiate_handover fleet_id, agent_id, to_agent
context_summary		 Trigger handover from primary to relief. Auto-triggered when watch limit is hit.
fire_alarm fleet_id, agent_id Wake a resting agent. Auto-fired by WhiteRoom's background timer when rest period ends.
get_handover fleet_id, agent_id Retrieve the compressed handover document for an agent. Requires the fleet's API key.
fleet_report fleet_id Full fleet status — agent counts, totals, savings, compliance, recent audit events.
list_fleets none List all active fleets. Used by the dashboard to auto-discover your fleet.

Limitations & Roadmap
What to know before using

WhiteRoom is early infrastructure. Here's what works, what doesn't, and what's coming.

⚠ Current Limitations

  • State is in-memory — restarts clear all agent registrations
  • Agents auto-re-register on next call after a restart
  • Gemini format not yet supported (OpenAI format only)
  • No horizontal scaling — single WhiteRoom instance
  • Watch/rest set to 10 minutes for testing (6 hours in production)
  • Handover context is LLM-generated — quality depends on task history richness

→ Coming Next

  • Persistent state via Redis or SQLite
  • Skill-based agent queuing and task routing
  • Hierarchical fleet governance (orchestrator + sub-agents)
  • Gemini and other LLM format support
  • npm package — npm install whiteroom
  • Python SDK — pip install whiteroom
  • WebSocket real-time dashboard updates
